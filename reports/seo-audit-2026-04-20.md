## 1. CRITICAL: Blog Off-Brand Content

### Finding
The blog contains **16+ robotics/humanoid posts** alongside AI receptionist content.
Sample off-brand posts found in crawl:
- "Humanoid Robot Cops Join Urban Forces"
- "Predictive Intelligence: Robots That Anticipate"
- "Humanoid Robots vs. Hype in Supply Chains"
- "Robotics Revolution in Motion: Top Breakthroughs from January 2026"
- "China's Robot Ambitions Expand Into New Markets"

### Why this hurts
Google measures topical authority — consistent coverage of a tight topic cluster (AI
receptionist for SMBs) builds ranking power for all related keywords. Mixing in
industrial robotics content signals to Google that this domain covers "robotics" broadly,
not "AI answering services" specifically. The result: lower authority for your actual
target keywords.

### Fix (do this week)
1. Redirect all robotics posts to a new subdomain (`robotics.newo.ai`) or noindex them
2. Never publish off-topic content to the main blog again
3. Use the blog exclusively for: AI receptionist, vertical use cases (dental/HVAC/
   restaurant), call answering, appointment booking, customer service automation

## 4. Schema Markup Gaps

### Current State
The homepage has basic WebPage + WebSite + Organization schema (from Yoast/RankMath).
Vertical pages have no Product, Service, or FAQ schema.

### What's missing and why it matters

**Service schema** on all vertical pages — tells Google exactly what service is offered,
who it's for, the price range. Enables rich snippets.

**FAQPage schema** — the FAQ section on dental/HVAC/restaurant pages is unstructured.
Adding FAQPage schema can capture "People Also Ask" boxes, doubling your SERP real estate.

**Product schema on Pricing page** — enables price display directly in search results.

### Implementation (add to each vertical page's `<head>`)

```json
// Dental page — Service schema
{
  "@context": "https://schema.org",
  "@type": "Service",
  "name": "Dental AI Receptionist",
  "provider": {
    "@type": "Organization",
    "name": "Newo.ai"
  },
  "serviceType": "AI Receptionist for Dental Practices",
  "areaServed": "United States",
  "description": "AI receptionist that answers dental practice calls 24/7, books appointments, and handles patient inquiries automatically.",
  "offers": {
    "@type": "Offer",
    "price": "99",
    "priceCurrency": "USD",
    "priceSpecification": {
      "@type": "UnitPriceSpecification",
      "price": "99",
      "priceCurrency": "USD",
      "billingDuration": "P1M"
    }
  }
}
```

```json
// FAQ schema example (add for each Q&A section)
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "How much does a dental AI receptionist cost?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Newo.ai dental AI receptionist plans start at $99/month, compared to $3,000+/month for a full-time front desk employee."
      }
    },
    {
      "@type": "Question",
      "name": "Is the AI HIPAA compliant for dental practices?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Yes. Newo.ai processes patient call data in compliance with HIPAA guidelines and does not store sensitive health information."
      }
    }
  ]
}
```
